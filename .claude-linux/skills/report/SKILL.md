セッションの作業内容を my-vault の日次ノートに追記する。

## なぜこの skill があるか

Claude Code セッションは流れていく。コミット履歴や diff には残らない「**何を決めたか・なぜそうしたか**」が、時間が経つと思い出せなくなる。
daily ノートに薄く残しておくことで、後から `[[ファイル名]]` でつなぎ直せる文脈の入口を作る。

形式の整合性より「**未来の自分が読み返せること**」が優先。

## コンテキスト

- 今日の日付: !`date +%Y-%m-%d`
- 現在時刻: !`date +%H:%M`
- 作業ディレクトリ: !`pwd`

## 手順

### 1. セッション内容を要約

会話を振り返って、**未来の自分が「あの日何やったっけ」と1秒で思い出せる**箇条書きを書く。

書くべきこと:
- 何を**決めた**か（採用した方針・捨てた選択肢）
- 何を**作った**か（ファイル・skill・hook・ノート）
- 何を**学んだ**か（仕様の発見、誤解の修正）
- 関連 my-vault ノートがあれば `[[ファイル名]]` でリンク

書かないこと:
- コミット履歴や diff で分かる技術的詳細（変更行数、関数名の羅列など）
- セッションの実況中継（「まず X を確認、次に Y を実行...」）

点数の目安は 3〜7 点。内容に応じて増減して良い（短いセッションは 2 点でも、濃い 1 日は 10 点でも構わない）。

### 2. 日次ノートのパス

パス規則は `~/workspace/my-vault/CLAUDE.md` に従う。**ハードコードせず、CLAUDE.md を読んで現在の構造を確認すること**（過去にflat→年/月分割のような変更があったため）。

### 3. 追記ルール

- 既存ノートがあれば末尾に `## Claude Code セッション（HH:MM）` セクションで追加
- ノートが無ければ vault のテンプレート（`# YYYY-MM-DD` + `## やったこと` / `## メモ・気づき` / `## リンク`）で新規作成し、その後セクション追加
- **同じ時刻のセクションが既にある場合**（SessionEnd フックの `[自動]` 版と重複など）は上書きせず、ユーザーに「追記するか・スキップするか」を確認

### 4. 確認

書いた内容をユーザーに提示し、修正があれば反映する。
